operating rules

What keeps diagrams usable

  • Lane = owner, system, or machine. Make the labels obvious.
  • Every box is verb-led and short. Decisions get diamonds only when there is a branch.
  • Show handoffs explicitly. If two teams touch the work, the line crosses lanes.
  • Default export is draw.io `design` XML with orthogonal connectors.
format reference

Design XML spine

- Root: mxfile > diagram > mxGraphModel > root
- Lanes: strip-*, body-*, label-*
- Nodes: start, end, step-*, decision-*, note-*
- Edges: edge-start, edge-approve-yes, edge-end
- Style: orthogonalEdgeStyle + endArrow=block
- Layout order: title → lanes → steps → connectors → notes
